# 苍穹研发问题原因分析

## 适用场景

- 生产环境故障排查
- 系统性能问题分析
- 代码Bug根因定位
- 部署/发布问题诊断
- 异常告警分析

## 分析流程

### 第一步：问题定义

明确以下信息：
- **现象描述**：用户看到什么错误？系统表现如何异常？
- **影响范围**：哪些功能/模块/用户受影响？
- **时间信息**：何时开始？是否间歇性？
- **触发条件**：什么操作会导致问题？

### 第二步：数据收集

根据问题类型收集对应数据源：

**日志数据**：
- 应用日志（error/warn级别）
- 系统日志（/var/log/）
- 访问日志（nginx/apache）
- 数据库慢查询日志

**监控数据**：
- 性能指标（CPU/内存/磁盘/网络）
- 链路追踪（Jaeger/SkyWalking）
- 告警历史
- 业务指标趋势

**代码/配置**：
- 近期代码变更（git log）
- 配置文件变更
- 依赖版本变化
- 环境差异对比

### 第三步：根因分析

使用 5 Whys 方法逐层深入：

```
问题现象
    ↓ 为什么？
直接原因
    ↓ 为什么？
中间原因
    ↓ 为什么？
根本原因
```

常见根因分类：
- **代码缺陷**：逻辑错误、边界条件、并发问题
- **配置问题**：参数错误、环境差异、配置漂移
- **资源问题**：内存泄漏、连接池耗尽、磁盘满
- **依赖问题**：第三方服务故障、版本不兼容
- **数据问题**：脏数据、数据迁移、索引缺失

### 第四步：验证假设

对每个可能原因进行验证：
- [ ] 复现问题
- [ ] 查看相关日志证据
- [ ] 检查监控指标关联性
- [ ] 对比正常/异常环境

## 常用命令参考

**日志搜索**：
```bash
# 按关键词搜索
grep -i "error\|exception" app.log

# 按时间范围搜索
awk '/2024-01-15 14:30/,/2024-01-15 15:00/' app.log

# 实时跟踪
tail -f app.log | grep ERROR
```

**性能分析**：
```bash
# 查看资源使用
top / htop

# 线程Dump
jstack <pid> > thread_dump.txt

# 内存分析
jmap -dump:format=b,file=heap.hprof <pid>
```

**Git分析**：
```bash
# 近期变更
git log --since="1 day ago" --oneline

# 文件变更历史
git log -p --follow <file>

# 某行代码最后修改
git blame <file>
```

## 分析技巧

1. **时间线对齐**：将日志、监控、告警按时间轴对齐，寻找关联性
2. **对比分析**：对比正常环境与异常环境的差异
3. **分层排查**：从应用层 → 框架层 → 系统层 → 网络层逐层排查
4. **假设驱动**：先提出假设，再寻找证据验证或证伪
5. **关注变化**：问题发生前有什么变更？变更往往是问题的源头
